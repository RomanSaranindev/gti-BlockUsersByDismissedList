# гти_БлокировкаУволенныхПользователей

Внешняя обработка 1С для автоматической блокировки пользователей из списка уволенных сотрудников, поступающего по электронной почте.

---

## Назначение

Каждый день HR-система отправляет письмо со списком уволенных сотрудников. Администратор сохраняет письмо как `.txt` файл в указанный каталог. Обработка читает файл, находит соответствующих пользователей в базе 1С и выполняет блокировку.

---

| Параметр | Значение |
|----------|----------|
| Платформа | 1С:Предприятие 8.3.27.1859 |
| Конфигурация | БИТ: Строительство (БИТ.ФИНАНС) КОРП 3.0 |
| Режим работы | Клиент-серверный |
| Файл обработки | `build/гти_БлокировкаУволенныхПользователей.epf` |

## Формат входящего файла

Администратор получает письмо от HR-системы, сохраняет его как `.txt` (Outlook → «Сохранить как» → Текст) в настроенный каталог.

**Пример содержимого файла:**

```
От:        Сервисы личного кабинета <hrplatform@areal.ru>
Отправлено: 14 июля 2026 г. 10:02
Кому:      Администратор 1С
Тема:      Сервисы личного кабинета | Информация об уволенных сотрудниках

Уволенные сотрудники от: 2026-07-13

ФИО: Иванов Иван Иванович,
Должность: Инженер-программист,
Отдел: Отдел автоматизации,
Организация: ООО "Пример",
Дата приема: 01-03-2022,
Дата увольнения: 13-07-2026,
Вид трудоустройства: Основное место работы

ФИО: Петров Пётр Петрович,
Должность: Руководитель,
Отдел: Дирекция,
Организация: ООО "Пример",
Дата приема: 15-06-2020,
Дата увольнения: 13-07-2026,
Причина увольнения: Перевод
Вид трудоустройства: Основное место работы
```

Поддерживается два формата признака перевода:
- Поле `Причина увольнения: Перевод`
- Блок `Другие места работы:` с непустым содержимым

## Интерфейс обработки

### Вкладка «Рабочая область»

![Рабочая область](docs/screen_work.png)

Содержит таблицу найденных сотрудников и кнопки управления:

| Кнопка | Действие |
|--------|----------|
| Загрузить список | Читает последний `.txt` файл из настроенного каталога |
| Выполнить блокировку | Блокирует всех отмеченных пользователей |
| Отметить все | Ставит галочку всем найденным пользователям (кроме переведённых) |
| Снять все отметки | Снимает все галочки |

**Колонки таблицы:**

| Колонка | Описание |
|---------|----------|
| ✓ | Отметка для блокировки |
| ФИО из файла | Как записано в файле выгрузки |
| Пользователь 1С | Найденная ссылка на справочник Пользователи |
| Должность | Из файла выгрузки |
| Организация (файл) | Строка из файла выгрузки |
| Организация (1С) | Ссылка на `Справочник.Организации` — найдена по наименованию в кавычках |
| Дата приёма | Из файла выгрузки |
| Дата увольнения | Из файла выгрузки |
| Визы | Список названий виз из `РС бит_ПраваУстановкиВиз` |
| Роли | Список ролей пользователя ИБ |
| Перевод | Признак перевода в другую организацию |
| Орг. после перевода | Ссылка на организацию перевода из `Справочник.Организации` — извлекается из поля «Другие места работы» |
| Другие места работы | Полная строка о переводе из файла |
| Статус | Результат после блокировки |

### Вкладка «Настройки»

| Параметр | Описание |
|----------|----------|
| Каталог выгрузки | Путь к папке, куда сохраняются `.txt` файлы |
| Список администраторов | Пользователи 1С, которым придёт HTML-лог на почту |

Настройки сохраняются в `ХранилищеОбщихНастроек` и восстанавливаются при следующем запуске.

## Логика блокировки

```
Пользователь найден в базе?
├── НЕТ  → пропустить, записать в лог серым
└── ДА
    ├── Переведён в другую организацию?
    │   └── ДА  → только комментарий в карточке, не блокировать (жёлтый в логе)
    └── НЕТ
        ├── Есть визы (бит_ПраваУстановкиВиз) или роли ИБ?
        │   └── ДА  → Недействителен = Истина + сброс аутентификации ИБ
        │             + пометка в логе (оранжевый — требует ручной проверки)
        └── НЕТ → Недействителен = Истина + сброс аутентификации ИБ (зелёный в логе)
```

**Действия при блокировке:**
1. Устанавливается флаг `Недействителен = Истина` в справочнике Пользователи
2. В пользователе ИБ сбрасываются все методы аутентификации (стандартная, ОС, OpenID)
3. В поле `Комментарий` карточки пользователя записывается штамп:

```
14.07.2026 10:30:00 Саранин Роман Геннадьевич блокировка пользователя
выполнена автоматически внешней обработкой Уволен от: 13-07-2026
```

## Поиск пользователей

Обработка ищет пользователя в два этапа:

1. **По физическому лицу** — `Справочник.Пользователи.ФизическоеЛицо.Наименование`
2. **По наименованию пользователя** — поле «Полное имя» после очистки постфикса

**Очистка постфиксов:** заглавные слова в конце ФИО (коды организаций) отсекаются автоматически.  
Пример: `Иванов Иван Иванович ГТИ` → ищется как `Иванов Иван Иванович`

## Уведомления администраторам

После выполнения блокировки на почту всем администраторам из настроек отправляется HTML-письмо с таблицей результатов:

- 🟢 Зелёный — заблокирован штатно
- 🟠 Оранжевый — заблокирован, но есть визы/роли (требует внимания)
- 🟡 Жёлтый — переведён в другую организацию, не заблокирован
- ⚫ Серый — не найден в базе 1С

**Колонки HTML-таблицы в письме:** ФИО, Пользователь 1С, Организация, Должность, Дата увольнения, Визы, Роли, Переведён, **Орг. перевода**, Статус.

Отправка через `РаботаСПочтовымиСообщениями.ОтправитьПочтовоеСообщение()` с системной учётной записью электронной почты.

## Архитектура (клиент-серверный режим)

В клиент-серверном режиме 1С файловые операции на сервере видят только файловую систему сервера. Поэтому реализована следующая схема:

```
Клиент                                    Сервер
──────────────────────────────────        ──────────────────────────────
НайтиФайлы() — ищет в папке              
клиента                                   
                                          
ЧтениеТекста() — читает файл              
с автоопределением кодировки              
(UTF-8 → CP1251 → системная)             
                                          
ПоместитьВоВременноеХранилище() ────────► ПолучитьИзВременногоХранилища()
                                          РазобратьТекстВыгрузки()
                                          ЗаполнитьТаблицуСотрудников()
                                          ВыполнитьБлокировкуСервер()
```

## Установка и запуск

1. Скопировать файл `build/гти_БлокировкаУволенныхПользователей.epf` на компьютер администратора
2. Открыть в 1С: **Файл → Открыть** → выбрать `.epf` файл
3. На вкладке **Настройки**:
   - Указать каталог, в который сохраняются `.txt` файлы выгрузки
   - Добавить администраторов-получателей лога
   - Нажать **Сохранить настройки**
4. Ежедневная работа:
   - Сохранить письмо из Outlook в настроенный каталог (Файл → Сохранить как → Текст)
   - Открыть обработку → нажать **Загрузить список**
   - Проверить найденных пользователей, при необходимости снять отметки
   - Нажать **Выполнить блокировку**
