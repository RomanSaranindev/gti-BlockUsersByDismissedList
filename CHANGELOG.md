# Журнал разработки — гти_БлокировкаУволенныхПользователей

---

## [1.1.0] — 2026-07-17

### Добавлено

#### Организация из базы 1С
- Новый реквизит ТЧ `ОрганизацияВБазе` (`CatalogRef.Организации`) — ссылка на организацию в базе 1С, найденную по данным файла выгрузки
- Колонка **«Организация (файл)»** — строка из файла выгрузки (переименована из «Организация»)
- Колонка **«Организация (1С)»** — ссылка из `Справочник.Организации`

#### Организация после перевода
- Новый реквизит ТЧ `ОрганизацияПослеПеревода` (`CatalogRef.Организации`) — организация, в которую переведён сотрудник, извлекается из поля «Другие места работы»
- Колонка **«Орг. после перевода»** — расположена после колонки «Перевод»

#### Алгоритм поиска организации (`НайтиОрганизациюПоНаименованиюИзФайла`)
- Из строки вида `Общество с ограниченной ответственностью "ГЕОТЕХ ИННОВЭЙШН"` извлекается краткое наименование в кавычках (`ГЕОТЕХ ИННОВЭЙШН`)
- Поддерживаются прямые `"..."` и типографские `«...»` кавычки
- Поиск по подстроке в `Справочник.Организации` без чувствительности к регистру
- Если кавычек нет — поиск по полному наименованию из файла

#### Алгоритм извлечения организации перевода (`ИзвлечьОрганизациюИзДругихМестРаботы`)
- Строка «Другие места работы» имеет формат: `ФИО, Должность, Отдел, Организация`
- Организация — всё начиная с 4-го элемента (корректно обрабатывает запятые внутри названия)
- При нескольких записях через `; ` берётся первая

#### HTML-отчёт администраторам
- Добавлена колонка **«Организация»** — основная организация сотрудника (из базы 1С или строка из файла как fallback)
- Добавлена колонка **«Орг. перевода»** — организация после перевода (из базы 1С → строка «Другие места работы» → пусто)

### Исправления
- Опечатка `СПАСИМВОЛЫ` → `СПЕЦСИМВОЛЫ` в тексте запроса (не поддерживалась в данной конфигурации базы, заменено на перебор через `СтрНайти`)

Внешняя обработка 1С для автоматической блокировки уволенных пользователей на основе ежедневной email-выгрузки.

**Платформа:** 1С:Предприятие 8.3.27.1859  
**Конфигурация:** БИТ: Строительство (БИТ.ФИНАНС) КОРП 3.0  
**Файл обработки:** `build/гти_БлокировкаУволенныхПользователей.epf`  
**Исходники:** `src/гти_БлокировкаУволенныхПользователей/`

## [1.0.0] — 2026-07-15

### Реализован функционал

#### Форма обработки
- Две вкладки: **Рабочая область** и **Настройки**
- Кнопки: «Загрузить список», «Выполнить блокировку», «Отметить все», «Снять все отметки»
- Таблица найденных сотрудников с колонками:
  - `✓` — флаг блокировки (галочка)
  - ФИО из файла
  - Пользователь 1С
  - Должность
  - Организация
  - Дата приёма / Дата увольнения
  - Визы (список названий через запятую)
  - Роли (список названий через запятую)
  - Перевод (галочка)
  - Другие места работы
  - Статус

#### Вкладка «Настройки»
- Поле выбора каталога выгрузки (с кнопкой «...»)
- Таблица администраторов-получателей лога (пользователь + адрес почты)
- Кнопка «Сохранить настройки» — сохраняет в `ХранилищеОбщихНастроек`

#### Парсер файла выгрузки
- Читает `.txt` файл на **клиенте** (решение для клиент-серверного режима)
- Автоопределение кодировки: UTF-8 → Windows-1251 → системная
- Поддержка заголовков письма Outlook (`От:`, `Отправлено:`, `Тема:` и др.) — пропускаются
- Распознаёт **два формата признака перевода**:
  - `Другие места работы:` с непустым содержимым (старый формат)
  - `Причина увольнения: Перевод` (новый формат)
- Очистка ФИО от постфиксов-организаций (заглавные слова в конце: `ГТИ`, `МСК` и т.д.)

#### Поиск пользователей
- **Приоритет 1:** по физическому лицу (`Пользователи.ФизическоеЛицо.Наименование`)
- **Приоритет 2:** по наименованию пользователя (после очистки постфикса)
- Запрос без условия `ЭтоГруппа` (справочник неиерархический)

#### Логика блокировки

| Условие | Действие | Цвет в логе |
|---------|----------|-------------|
| Нет виз и ролей | `Недействителен = Истина` + сброс аутентификации ИБ | Зелёный |
| Есть визы **или** роли | `Недействителен = Истина` + сброс аутентификации ИБ + пометка в логе | Оранжевый |
| Переведён в др. организацию | Только комментарий, ничего не блокируется | Жёлтый |
| Не найден в базе | Пропуск с записью в лог | Серый |

#### Комментарии в карточке пользователя
Шаблон: `ДД.ММ.ГГГГ ЧЧ:ММ:СС ИмяАдминистратора блокировка пользователя выполнена автоматически внешней обработкой Уволен от: ДД-ММ-ГГГГ`

#### Проверка виз и ролей
- Визы: запрос к `РегистрСведений.бит_ПраваУстановкиВиз` — возвращает список названий
- Роли: `ПользователиИнформационнойБазы.НайтиПоУникальномуИдентификатору()` — список имён ролей ИБ

#### Отправка лога
- HTML-таблица с цветовой индикацией строк
- Отправка через `РаботаСПочтовымиСообщениями.ОтправитьПочтовоеСообщение()` с системной учётной записью
- Тело письма в формате HTML (`ТипТекста = "HTML"`)
- Получатели: список из вкладки «Настройки» (пользователь 1С → адрес из контактной информации, или явно указанный адрес)

## Исправленные ошибки в процессе разработки

| # | Ошибка | Причина | Решение |
|---|--------|---------|---------|
| 1 | `Ожидается константа типа Число, Строка...` | `Разделитель = Символы.ПС` в параметре по умолчанию | Заменено на `""` |
| 2 | `Переменная не определена (ФорматыТелаЭлектронногоПисьма)` | Несуществующий тип | Заменено на структуру с `ТипТекста = "HTML"` |
| 3 | `Переменная не определена (ИнформационныеБазы)` | Несуществующий объект | Заменено на `ПользователиИнформационнойБазы.НайтиПоУникальномуИдентификатору()` |
| 4 | `Метод объекта не обнаружен (ЗагрузитьНастройки)` | Методы модуля объекта недоступны через `Объект.Метод()` во внешней обработке | Вся бизнес-логика перенесена в `&НаСервере` модуля формы |
| 5 | Форма отображается пустой | Тег `<PagesGroup>` не существует, отсутствие `<ExtendedTooltip>`, `<ContextMenu>`, служебных блоков таблиц | Form.xml перегенерирован через `form-compile` |
| 6 | `Поле не найдено "Пользователи.ЭтоГруппа"` | Справочник неиерархический, поле недоступно | Условие убрано из запроса |
| 7 | Файл не читается в клиент-серверном режиме | `НайтиФайлы()` на сервере видит файловую систему сервера, путь клиентский | Поиск и чтение файла перенесены на клиент, текст передаётся через `ПоместитьВоВременноеХранилище()` |
| 8 | Загружается только последний сотрудник | `СтрРазделить(..., Ложь)` отбрасывал пустые строки-разделители блоков | Заменено на `СтрРазделить(..., Истина)` |
| 9 | Файл не читается (кодировка) | Outlook сохраняет `.txt` в CP1251, читали в UTF-8 | Добавлено автоопределение кодировки |
| 10 | XDTO-ошибка Form.xml | Несовместимый namespace `dcssch` и `<EditMode>Always</EditMode>` в версии 2.17 | Убран `dcssch`, убран `EditMode Always` |
